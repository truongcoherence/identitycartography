# Structuria — New Source Ingestion Criteria (Instruction Set)

**Status:** July 14, 2026. A generalized template, extracted from the StoryCorps Ingestion Policy, for evaluating any future external data source before building a source-specific policy. Deliberately left unfilled for AHRQ PSNet and NASA APPEL — meant to be completed against each source directly, not drafted in the abstract.

**Why this document exists:** this project was designed for constrained environments — institutional, high-stakes, multi-party. It follows that it should be tested against accounts from that same kind of environment, not just against whatever external data happens to be easiest to reach. This document exists because that realization came late, and shouldn't be lost the next time this document gets revised for something else.

---

Every new source's ingestion policy should answer these questions explicitly, in this order:

1. **Licensing.** Is this source legally clear to use — public domain, explicit permission granted, or a pending request? Distinct from every question below; resolving this does not resolve any of them.

2. **Source-side anonymization status.** Are the people in this material already de-identified by the publisher, or would this project need to apply its own anonymization work before any public use? This sets the default Exposure Tier a case from this source should start at — not assumed favorable just because licensing is clean.

3. **Environment-type match.** Is this source drawn from institutional, high-stakes, constrained environments, or from personal, reflective narrative? Tonight's actual results say this matters: reflective personal narrative has so far failed adversarial reframe uniformly. Set expectations about likely yield before starting, not after.

4. **Archive status.** Living and growing, or fixed and frozen as of a known date? Changes how traversal and long-term planning should work.

5. **Selection and traversal method.** Systematic, full-read, one-per-day or similar — the same discipline already established, adapted to this source's actual format and size.

6. **Domain-coverage expectation.** Which of the five IDPs, the Meta-Concept Layer, and the Architecture of Signal and Outcome is this source likely to serve well, and which is it not expected to serve at all? State this before starting, so an absence of hits in an off-target domain isn't mistaken for a failure of the source or the method.

7. **Expected base rate, stated in advance.** Given what this source actually is, roughly how many valid cases should be expected out of how many reviewed? A rate matching this expectation is confirming evidence the screen is working correctly, not a disappointing result — and a rate that beats expectation is exactly the kind of outlier worth the closest attention, precisely because the screen has already shown it can reject what should be rejected.

8. **Publication gate.** Does public use of this source require a separate, explicit commitment beyond this project's own internal Publication & Anonymization Standard — the way StoryCorps does — or does the source's own public-domain status make that unnecessary? Confirm rather than assume either way.

Fill in all eight for a given source before it enters active use. Two sources with clean answers to #1 (licensing) can look very different once #2 through #8 are actually worked through — treat the full set as required, not the licensing question alone.
