# Structuria — Limitations and Invitation to Extend

**Status:** July 13, 2026. Written as its own document, not a caveat buried inside another one, because the questions it answers are load-bearing for anyone deciding how much weight to put on this work — including the author.

---

## 1. What This Project Actually Claims

Structuria (Identity Cartography) proposes a set of structural patterns — five domains, each with tested invariants — describing how identity and behavior respond to constraint, independent of the motivation, personality, or moral character of the person involved. It does not claim to be a mature scientific theory. It claims to be an honestly-graded, early-stage framework, built through a documented method, with its own confidence levels stated per claim rather than uniformly. Where the evidence is thin, this document says so in numbers, not in hedging language.

---

## 2. The Single-Coder Problem

Every classification decision in this corpus — which domain a case belongs to, whether it confirms, refines, or breaks an existing invariant — has been made by one analytical process: the author, working with Claude. The Case Validity Checklist and Pattern Comparison Protocol both require a blind second pass before high-stakes decisions, but that second pass has also always been Claude. This is a real, unresolved limitation, not a solved one. The project's own founding event — three different AI systems, given identical rules, producing three different coherent classifications of the same field note — is direct evidence that a single analytical process, however disciplined, is not sufficient to guarantee a classification is correct rather than merely internally consistent. Nothing in this project's methodology closes that gap. It can only be closed by genuinely independent analysis, from outside this project, which has not yet happened.

---

## 3. Narrow Evidentiary Base

Of the 36 cases currently in the Case Registry, the overwhelming majority are drawn from one person's life and one professional field — healthcare technology management within the U.S. federal healthcare system. Stewardship's restoration required an explicit methodological departure (sourced from adjacent professional material rather than the lived parenting continent the domain was originally named for) specifically because this narrowness is already visible from inside the project, not just in principle. Whether these patterns generalize outside engineering, healthcare, and federal institutional contexts is untested.

**A second, independent cost of this narrowness, distinct from generalizability: anonymization strength is currently capped by dataset size.** Per the Publication & Anonymization Standard's baseline default, case narratives replace the protagonist's identity with a generic label ("the subject"). That substitution only provides real protection to the extent other plausible candidates exist behind the same label — a principle from data privacy work known as k-anonymity. With this corpus drawn almost entirely from one life, "the subject" currently has exactly one real candidate behind it in nearly every case, which offers close to no protective ambiguity for that person specifically, even though it still fully protects every other party in each account. External case collection is therefore not only a generalizability need but an anonymization one: as genuinely diverse cases accumulate, the same label stops reliably pointing at any one person at all.

**A related, distinct pattern: this corpus's intake has run on intensity, not on domain-specific search terms — for a specific, defensible reason, with a specific, real consequence.** The author's actual selection criterion for candidate life events has been "conflict, failure, or high energy interaction," not a hunt for material matching any invariant. This avoids the confirmation-bias risk the Case Validity Checklist was built to catch. It does not avoid a different one: low-intensity, successfully-resolved, or otherwise undramatic instances of the same structural mechanisms are structurally under-represented, simply because they are harder to notice as instances of anything at all. The author's own account of why is a genuine calibration claim, not an excuse: a high-intensity case is frequently what teaches a person to recognize the shape of a pattern in the first place, after which fainter instances of the same pattern become visible retrospectively and going forward. This is checkable against the existing corpus rather than merely asserted. In both Judgment and Boundaries, a high-stakes genesis case did precede, and plausibly enabled, the later recognition of much lower-stakes instances of the same domain (a weekend-plans indecision; a workplace authority-room moment). If this pattern holds as more cases are gathered, it should be tracked explicitly as a feature of how this research process itself develops — a calibration curve, not merely a defect — while the underlying imbalance in the current evidentiary base remains real and worth correcting going forward, now that the anchor cases have done their calibrating work.

## 4. Evidentiary Thinness, Stated in Numbers

This corpus currently contains 23 formally restored invariants and constraints across five domains. Of those, 7 (30%) rest on a single source case. One of those single cases — the timer incident — independently backs four separate constraints in Judgment; despite reading as four claims, it is one underlying data point. The stated bar for promoting a candidate pattern to formal invariant status is two independent, non-reused sources — a floor observed to work in this project so far, not a principled saturation criterion in the sense qualitative research methodology normally means by that term.

By exposure tier: 22 of 36 cases (61%) are Tier A, 7 (19%) are Tier B, and 6 (17%) are Tier C. That 17% is not evenly distributed — it concentrates specifically in Peril, whose best-evidenced invariant is Tier C across all three of its sources with no lower-exposure alternative available, and in Boundaries' two highest-stakes relational cases. The domain with the thinnest evidentiary base and the domain with the highest exposure risk are the same domain. This is treated in this project as a real tension requiring more work, not as a coincidence to explain away.

---

## 5. Retrospective Reconstruction Risk

A meaningful share of this corpus's highest-stakes cases — including the single most-cited case in Peril — are high-stakes retrospective accounts: real, but recalled after the outcome was already known, carrying reconstruction risk in a way that routine, low-stakes field observation does not. This project distinguishes the two categories in its own working practice but has not yet systematically weighted them differently in how evidentiary strength gets reported.

---

## 6. The Unresolved Realism Question

This project holds its invariants to a falsifiability standard — Substitution, Independent Derivation, Circularity, Compression, Adversarial Reframe — because a claim that can never fail does no discriminating work, regardless of whether it is "really" true in a deeper sense. Whether these patterns are genuinely mind-independent structural regularities, closer to how physical law is usually understood, or context-dependent patterns partly co-constructed by the observer, closer to how constructivist qualitative research understands its own categories, is a live, open question this project has not settled and does not claim to have the internal means to settle. Readers should not assume the word "physics" in this project's own vocabulary is making a stronger ontological claim than the evidence in §4 above can currently support.

**A separate point, worth stating on its own rather than folded into the caution above: the vocabulary itself has real, defensible value independent of how that ontological question eventually resolves.** Calling something "insulation failure, contact, revelation" instead of "you should have been more careful" is not a stylistic preference. Ordinary narrative and psychological language is saturated with intentionality and character-attribution — English defaults to describing what someone *is* or *failed to be*, not what structurally happened around them. Physics-adjacent vocabulary — pressure, load, forces, contact — supplies a genuinely different semantic field that doesn't carry that freight by default. This is not unique to this project; safety engineering and aviation incident analysis rely on comparably mechanical, blame-resistant vocabulary for exactly this reason, treating language choice as a structural tool rather than decoration. Whether or not this project's claims are ever shown to be physics in a stronger sense, describing events in terms of pressure and load rather than character and fault is doing real work — operationalizing Law 01 in the language itself, not just in the argument. That value does not depend on the epistemic status the rest of this section leaves open.

---

## 7. Incomplete Domains

Peril was restored with a substantially thinner and higher-exposure evidentiary base than Judgment, Boundaries, Cohesion, or Stewardship, and remains the newest, least battle-tested domain in the corpus. The Operator Bible — the layer describing Lens, Interpreter, Relief, Seismograph, Probe, and Compass — was flagged as structurally unstable at the outset of this project's audit and has never been formally re-tested against the same five-test standard applied to the domains. Claims resting on operator content should be read as more provisional than claims resting on the restored IDPs.

---

## 8. Publication-Layer Fidelity Cost

Any case moving from this project's internal working record toward public form will be paraphrased, generalized, or composited to protect the people who appear in it, per the Publication & Anonymization Standard. This is a deliberate, necessary tradeoff — but it means the public-facing evidentiary record is, by design, a lower-fidelity version of the internal one. A reader working only from published material is working from a version of this corpus that has already traded some precision for safety.

---

## 9. What Would Actually Extend or Challenge This Work

Not a request for agreement. A specific, actionable description of what a genuinely useful contribution looks like:

- **Applied in a different field or industry than healthcare technology management**, ideally by someone with no prior connection to this project.
- **Documented using this project's own Case Note format**, so it is comparable to existing entries rather than a freeform narrative.
- **Actively reporting disconfirming or non-fitting instances, not only confirming ones.** A case where an invariant's stated definition failed to hold, or where behavior under similar-looking constraint didn't match the predicted pattern, is more valuable to this project than another case that fits cleanly. Confirmation this project already has more of than it has independence.
- **Willing to be identified as an independent source**, distinct from this project's own author, so that any resulting corroboration is genuinely external rather than another instance of the single-coder problem in §2.

This project does not expect this to happen at scale, and does not need it to in order to be worth having built. One genuinely independent, well-documented case, from outside this project's own evidentiary base, would be stronger evidence than most of what currently exists in this corpus. That is the actual, realistic target — not volume, not recognition, just enough independent contact with the claim to know whether it holds outside the one life it was built from.
